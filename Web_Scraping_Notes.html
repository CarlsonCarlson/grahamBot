General notes

there are 2 selectors, xpath and css return selectors
selectorMethod().get() gives you a string of the first one found selectorMethod().getall() gives you a list of strings

Using CSS to find elements

response.css('h3::text').get() -> gives you the text of the first h3
response.css(h3::text)[1].get() -> this gives you the second h3 and so on
response.css('title::text').get() -> this gives you the text of title
response.css('.CLASSNAME a').getall() -> gives you all of the links in the CLASSNAME (remember the . before the classname)
.CLASSNAME lets you get that class
after the classname you may do space then tag, to get a certain tag of a class example: '.CLASSNAME a'
::text is an option to take only text, it only goes at end of the single quotes

Using regular expressions to find things

instead of get or getall you can use re you need to use r-strings ie. (r'hello') these don't process escape sequences and can make regular expressions (you can look these up)
response.css('.CLASSNAME a').re(r'hi') -> this gets all the instances of hi in the selector
response.css('.CLASSNAME a').re(r'hi \+w') -> this gets you all instances of hi {space} ANYWORD
response.css('.CLASSNAME a').re(r'hi\+w') -> this gets you all instances of words that start with 'hi'
response.css('.CLASSNAME a').re(r'(\+w) hi (\+w')) -> this get you all words before or after hi
